## 🎯 Что этот проект делает вообще:

Это самая первая машина обучения алгоритм. Мы учим компьюетр на примерах понимать зависимость: **чем больше пробег у машины тем меньше она стоит**.

Мы ничего не говорим компьютеру какая формула. Он сам находит коэффициенты сам смотря на данные.

## 📊 Данные которые есть:

У нас есть файл `data.csv` с 24 машинами. Для каждой написано сколько километров она проехала и за сколько продали.

| Пробег | Цена |
|---|---|
| 240000 км | 3650 € |
| 139800 км | 3800 € |
| ... | ... |
| 22899 км | 7990 € |

Если нарисовать эти точки на графике - мы увидим что они примерно лежат на прямой линии.

---

## 🧮 Формула которую мы ищем:

Мы ищем прямую линию которая максимально точно лежит между всеми точками:

```
Цена = theta0 + theta1 * Пробег
```

Где:
- `theta0` = цена машины с нулевым пробегом (новая машина)
- `theta1` = сколько евро теряет машина за каждый километр пробега

✅ В нашем обученном проекте получилось:
```
theta0 = 8499.60  ✔️  новая машина стоит ~8500 евро
theta1 = -0.0214  ✔️  за каждый километр машина дешевле на 2.14 цента
```

## ⚡ Алгоритм градиентный спуск:

Это сердце всего проекта. Именно это ты не можешь просто взять из библиотеки.

### ❗️ Самое важное правило всего проекта:
> ❗️ НИКОГДА НЕ ОБНОВЛЯЙ ТЕТУ ПО ОДНОЙ❗️

❌ НЕПРАВИЛЬНО:
```python
theta0 = theta0 - tmp0
theta1 = theta1 - tmp1   ❌ тут theta0 уже новый!
```

✅ ПРАВИЛЬНО:
```python
tmp0 = ... считаем
tmp1 = ... считаем

theta0 -= tmp0
theta1 -= tmp1   ✅ ОБА НА ОСНОВАНИИ СТАРЫХ ЗНАЧЕНИЙ
```

> Это самая частая ошибка которую делают 90% людей. Именно на этом пиры и ловят.

### 📄 train.py
Это самая главная программа. Обучение.

#### 1. Нормализация данных:
Самое важное что не написано в сабжекте но без этого алгоритм не работает:

❌ Если мы даем алгоритму числа как есть (100000 км, 7000 евро), то градиентный спуск никогда не сойдется. Он будет прыгать и никогда не найдет минимум.

✅ Поэтому мы переводим все числа в диапазон от -1 до +1:
```python
km_norm = (km - среднее_пробега) / стандартное_отклонение
```

После того как мы обучили теты на нормализованных данных - мы потом переводим их обратно в реальные единицы. Именно поэтому predict.py вообще ничего не знает про нормализацию.

#### 2. Цикл градиентного спуска:
1. Начинаем с theta0 = 0, theta1 = 0
2. Для каждой итерации:
   3. Считаем ошибку для каждой точки
   4. Суммируем ошибки
   5. Корректируем теты немного в правильную сторону
   6. Повторяем 1000 раз

Количество итераций 1000 и learning rate 0.1 - это идеальные значения которые подобраны экспериментально.

---

### 📄 predict.py
Программа для предсказания. Она максимально простая:

1. Пытается загрузить теты из model.json
2. Если файла нет или он битый - использует 0 и 0
3. Запрашивает пробег у пользователя
4. Считает цену по формуле
5. Выводит результат

✅ Очень важная вещь: если цена получилась отрицательная - он пишет предупреждение но все равно выводит значение. Это правильное поведение.

---

### 📄 precision.py
Считает насколько хорошо наша линия подходит под точки.

Самый важный показатель это **R²**:
| R² | Что значит |
|---|---|
| 1.0 | Идеально подходит, все точки лежат ровно на линии |
| 0.7 | Очень хороший результат для этого датасета |
| 0.0 | Наша линия работает так же хорошо как просто среднее значение |
| <0 | Еще хуже чем просто среднее |

✅ У нас R² = 0.7330. Это очень хороший результат. Это значит что 73% изменения цены объясняется пробегом.

## ❓ Самые частые вопросы пиров:

### ❓ Почему у тебя нормализация? В сабжекте про нее ничего не написано!
✅ Потому что без нее градиентный спуск не работает. Сабжект не говорит что ты не можешь делать нормализацию. Он говорит только какие формулы ты должен использовать. Все правильно.

### ❓ Почему ты используешь 0.1 learning rate и 1000 итераций?
✅ Потому что это оптимальные значения. Если сделать больше 0.1 - алгоритм разойдется. Если меньше - будет слишком долго сходиться. 1000 итераций это ровно столько сколько нужно чтобы он сконвергировал полностью.

### ❓ Почему ты не используешь numpy?
✅ Потому что тогда ты ничего не поймешь. Цель этого проекта не сделать линейную регрессию. Цель понять как она работает изнутри.
